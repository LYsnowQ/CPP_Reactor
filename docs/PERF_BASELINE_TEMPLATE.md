## 3. 术语解释（看日志时对照）
- `QPS`：每秒完成的请求数，衡量系统吞吐能力（越高越好，但要结合延迟看）。
- `Avg Latency`：请求平均耗时（微秒/毫秒），衡量整体响应速度（越低越好）。
- `Max Latency`：窗口内观测到的最大耗时，反映抖动与尾延迟风险。
- `Error Rate`：失败请求占比，衡量稳定性（越低越好）。
- `Active Connections`：当前活跃连接数，反映连接负载与资源占用。
- `Read Bps / Write Bps`：每秒读/写字节数，衡量网络流量规模与方向。
- `2xx / 4xx / 5xx`：HTTP 状态码分桶，分别对应成功、客户端错误、服务端错误。
