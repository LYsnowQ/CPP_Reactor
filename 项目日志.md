## 2026-04-22 18:27:24

- 版本：0.3.0。
- 当前进度：
  - `CoreStatus`：新增统一状态码定义，事件循环与分发器返回值统一到 `StatusCode`。
  - `Dispatcher`：新增分发器工厂与类型映射（`epoll/poll/select`），支持按类型创建后端。
  - `EventLoop`：新增 `addTask(int fd, ChannelOP)` 路径，任务队列改为局部队列批处理，`socketpair` 唤醒改为非阻塞读写并补充错误事件分支。
  - `Epoll/Poll/SelectDispatcher`：分发返回值统一，`EINTR` 处理补齐，错误事件映射到 `kErrorEvent`。
  - `TcpServer/IOThreadPool`：支持按分发器类型创建 worker loop，补充 `start/stop` 防重复处理，`accept` 失败与空 loop 场景增加保护路径。
  - `TcpConnection`：读路径接入 `HttpRequest::parseRequest`，写路径默认响应联通，关闭路径统一投递 `DELETE` 任务。
  - `Buffer/HttpRequest/HttpResponse`：查找与切片边界处理完善，请求行/请求头解析流程收敛，就绪位清理逻辑修正。
- 待完成内容：
  - `main.cpp` 与 C++ 服务器入口链路的统一切换。
  - `TcpServer` 主/子线程模式分支继续收敛。
  - `TcpConnection` 与连接容器回收策略闭环（连接生命周期最终收口）。
  - HTTP 响应组包与文件发送路径接入新 C++ 响应对象闭环。
- 行进目标：先完成运行入口统一与连接回收闭环，再推进 HTTP 响应链路全流程打通。
- 提交记录：对比上一条记录，新增状态码体系、分发器工厂、多后端可选能力与事件错误路径处理，连接与请求解析链路继续向 C++ 主路径收敛。
